| # | File | Issue | Fix Applied |

| 1 | `modules/filler.py` | `from reportlab.lib.units import pt` — `pt` does not exist in this module | Removed unused import; ReportLab natively uses points |
| 2 | `main.py` | `@app.on_event("startup")` is deprecated in FastAPI 0.111+ | Replaced with `@asynccontextmanager` lifespan pattern |
| 3 | `main.py` | `upd.dict()` deprecated in Pydantic v2 | Replaced with `upd.model_dump()` with fallback |
| 4 | `modules/analyzer.py` | `_find_nearby_label()` only picked the closest single word, not full multi-word labels | Rewritten to collect all words in the row above the cell and concatenate |
| 5 | `modules/field_normalizer.py` | `normalize_label()` only fuzzy-matched whole label, failing on compound bilingual labels like "Full Name (as per IC) / Nama Penuh" | Added 4-stage matching: whole label → segment split on `/` and `()` → per-segment fuzzy → whole-label loose fuzzy |
| 6 | `modules/field_normalizer.py` | `address_line1` missing "Address Line 1" as an explicit synonym, causing it to match `address_line2` | Added "address line 1", "address 1", "alamat 1" etc. to `address_line1` synonyms |
| 7 | `requirements.txt` | All versions pinned to exact numbers, causing resolution conflicts on some systems | Changed to compatible-release `>=x,<y` ranges |

## 10. Known Limitations / Remaining TODOs

### Scanned PDF / OCR path
- **Status:** Code is complete; validated for structure only
- **Blocker:** Tesseract is not pre-installed in the default environment
- **Action:** On macOS: `brew install tesseract tesseract-lang poppler`
- **Impact:** Digital PDFs (the majority of bank forms used in production) work fully

### Chinese font rendering in PDF output
- **Status:** Code is complete with documented TODO
- **Blocker:** No Chinese TTF font bundled (would increase repo size by ~10 MB)
- **Action:** Drop `NotoSansSC-Regular.ttf` into `backend/fonts/` and uncomment 3 lines in `filler.py`
- **Impact:** ASCII/English content renders correctly; Chinese values will be skipped silently

### Production deployment
- The system is designed for `localhost` development
- For production: configure proper HTTPS, authentication, and set `NEXT_PUBLIC_API_URL`
